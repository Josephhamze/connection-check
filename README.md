# Connection Check

A small end-to-end test project showing a GitHub repository, a Vercel deployment, and live Supabase reads and writes.

## Environment

Copy `.env.example` to `.env.local` and set:

- `SUPABASE_URL`
- `SUPABASE_SECRET_KEY`
- `NEXT_PUBLIC_GITHUB_URL`

The Supabase secret is used only by server code and must never be exposed with a `NEXT_PUBLIC_` prefix.

## Database schema

```sql
create table public.connection_messages (
  id bigint generated by default as identity primary key,
  body text not null check (char_length(body) between 1 and 160),
  created_at timestamptz not null default now()
);

alter table public.connection_messages enable row level security;
```

The app uses a server-only key, so no public table policy is required.
